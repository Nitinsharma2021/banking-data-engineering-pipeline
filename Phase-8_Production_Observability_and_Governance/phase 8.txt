STEP 3 — DEPLOY CUSTOM METRICS LAMBDA
═══════════════════════════════════════

Script: step3_deploy_metrics_lambda.py

What this Lambda does:
  Triggered by EventBridge after each pipeline completion
  Reads all JSON files from s3://neo-bank-datalake/metadata/dq_results/
  Parses DQ rule results per table
  Publishes custom CloudWatch metrics:

    Namespace: AmerispriseBank/ETL
    Metrics:
      DQRulesPassed         (per table)
      DQRulesFailed         (per table)
      DQPassRate            (per table, percentage)
      RowsProcessed         (per zone: bronze, silver, gold)
      QuarantinedRows       (per table)
      JobDuration           (per Glue job, seconds)

  Also reads S3 to count rows in each silver/gold partition for the run_date
  Publishes RowsProcessed metric

Lambda function name:    neo-bank-metrics-publisher
Runtime:                 Python 3.11
Memory:                  512 MB
Timeout:                 5 minutes
IAM role:                AmerispriseBankMetricsLambdaRole
Trigger:                 EventBridge rule on Step Functions success

RUN: python step3_deploy_metrics_lambda.py

What gets created:
  - Lambda function with code
  - IAM role with policies for S3 read, CloudWatch put, Step Functions describe
  - EventBridge rule to trigger after pipeline succeeds


STEP 4 — SETUP ANOMALY DETECTION
══════════════════════════════════

Script: step4_setup_anomaly_detection.py

What it creates:
  CloudWatch Anomaly Detectors on these custom metrics:

  Metric: AmerispriseBank/ETL/RowsProcessed (per table)
    - Auto-learns baseline over 14 days
    - Alarms if today's count deviates > 2 standard deviations

  Metric: AmerispriseBank/ETL/JobDuration (per Glue job)
    - Auto-learns expected duration
    - Alarms if any job runs >2 std devs slower

Anomaly detectors are different from threshold alarms:
  Threshold:  "alarm if rows < 1000"  (manual setting, brittle)
  Anomaly:    "alarm if rows fall outside normal range"  (auto-learning)

For a banking pipeline that grows over time, anomaly detection is much
more reliable than fixed thresholds.

RUN: python step4_setup_anomaly_detection.py

NOTE: Anomaly detectors need 7-14 days of metric data to start working.
      Initially they will not fire alarms — this is expected.
